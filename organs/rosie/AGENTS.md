# AGENTS.md

## Cursor Cloud specific instructions

### Overview

**rosie** is a zero-external-dependency TypeScript library (receipt DAG + CSS-ingress QEC module). It runs on **Node.js >= 22** using `--experimental-strip-types` for native TypeScript execution. There is no build step, no web server, and no database.

### Dev tool installation

The repo has no committed `package.json`. The update script creates one and installs dev-only tooling (`typescript`, `eslint@8`, `@typescript-eslint/parser`, `tsx`, `@types/node`). These are only needed for type-checking and linting; tests themselves require only Node.js.

### Running tests

```bash
# Main test suite (CI approach) — 31 tests across 3 files
for test in tests/*.test.ts; do
  node --experimental-strip-types "$test"
done

# QEC ingress tests (7 tests) — uses extensionless imports, needs tsx
npx tsx src/qec/css_ingress.test.ts
```

**Gotcha:** `src/qec/css_ingress.test.ts` imports from `'./css_ingress'` (no `.ts` extension), so it cannot run with `node --experimental-strip-types`. Use `npx tsx` instead.

### Type-checking

```bash
npx tsc --noEmit --target ES2022 --moduleResolution bundler \
  --allowImportingTsExtensions --skipLibCheck --types node \
  src/*.ts src/qec/*.ts
```

No `tsconfig.json` exists; CI uses the flags above. The `--types node` flag is required because `@types/node` is installed locally (not globally).

### Linting

```bash
npx eslint --ext .ts src/ tests/
```

The repo uses `.eslintrc.json` (ESLint v8 config format), so ESLint v8 must be installed (not v9+). Existing warnings in `src/qec/qec_lineage.ts` (unused vars) are pre-existing.

### Important notes

- `node_modules/`, `package.json`, and `package-lock.json` are generated by the update script for dev tooling only; they are not part of the source repo and should not be committed.
- CI only runs `tests/*.test.ts` (not the QEC test in `src/qec/`). Both should be verified locally.
