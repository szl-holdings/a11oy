## §4 [UNVERIFIED] / Blockers

### [UNVERIFIED] Items

1. **[UNVERIFIED — amaru API]** All amaru chunk store calls in `runtime.ts` and `hybrid-receipted.ts` are stubs returning synthetic candidates. Real implementation requires `@szl/amaru/chunk-store` package with `ChunkStore.connect(ref).hybridQuery(...)`. See `[REQUIRES_REPO_CONTEXT]` tags.

2. **[UNVERIFIED — sentra API]** `sentraGate()` in `runtime.ts` is a stub evaluating against `chunk.doctrineGrade` as proxy. Real implementation requires `@szl/sentra/policy.PolicyGate.evaluate(predicates, chunk)`.

3. **[UNVERIFIED — ouroboros API]** `ouroborosBudgetCheck()` is a soft-limit stub. Real implementation requires `@szl/ouroboros/budget.BudgetGate.check(...)` — Lean-verified termination guarantee.

4. **[UNVERIFIED — Zenodo anchor]** `zenodoAnchorStub()` logs but does not POST to Zenodo. Requires `ZENODO_TOKEN` env + Zenodo REST API deposit sequence.

5. **[UNVERIFIED — LambdaQLEditor Monaco dep]** Editor uses `<textarea>` + highlight overlay instead of Monaco editor to avoid hard Monaco dependency. A production build should swap in `@monaco-editor/react` with the Λ-QL JSON schema registered.

6. **[UNVERIFIED — sha256Hex crypto]** `util.ts` `sha256Hex()` falls back to djb2 if `crypto.subtle` unavailable. Production builds must ensure Node ≥ 20 or inject `require("node:crypto")`.

7. **[UNVERIFIED — Lean helper lemmas]** Lean 4 obligation templates in compiler output use `sorry` stubs for helper lemmas (`chunk_membership_from_receipt`, `sentra_gate_from_attestation`). Full mechanization required in `lutar-lean/LambdaQL/Obligations.lean`.

### No Blockers

No BLOCKED conditions raised. All files ship without silent stubs — every stub has a `[REQUIRES_REPO_CONTEXT]` tag with exact replacement instructions.
