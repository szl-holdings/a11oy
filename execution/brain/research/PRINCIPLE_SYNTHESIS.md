# Public-source principle synthesis

The common useful pattern is explicit, scoped memory connected to durable source identity.
Graph, vector, and lexical representations are complementary indexes, not separate truths.
Temporal validity and contradiction handling matter as much as similarity. Durable workflow
systems reinforce that every externally visible change needs replayable state and idempotent
boundaries. Code-intelligence systems reinforce that repository symbols, commits, and
dependencies are high-value graph objects.

The Chinese open-source leader set adds a second useful pattern: separate the model, agent
runtime, serving contract, and accelerator kernels into independently admitted layers. Kimi's
agent/runtime and hybrid-attention work, Qwen's tool and MCP adapters, DeepSeek's kernel and
expert-parallel libraries, and MiniMax's serving guidance are recorded as revision-pinned
research inputs. SZL should express them as provider-neutral capabilities with conformance
tests, benchmark receipts, hardware-fit evidence, and explicit license gates. No model,
kernel, prompt, hidden reasoning, product identity, or benchmark claim is imported by
default.

SZL adopts those general principles through an original tenant-bound schema, epistemic
status model, admission contract, propagation transaction, deletion receipt, and honest
capability manifest. No external product identity, private implementation, prompt, interface,
or unverified benchmark claim is copied.
